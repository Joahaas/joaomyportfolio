The Challenge

Users struggled to manage and analyze large volumes of AI-generated content. The existing interface lacked proper organization tools and made it difficult to track conversation history and insights across multiple sessions.

Key Problems Identified

  • Context overload in long prompts
  • No easy way to save or retrieve key insights
  • Lack of tools to remix or build on ideas
  • Inefficient, disorganized AI workflows

The Goal

Streamline the user’s interaction with ChatGPT by enabling better organization, retrieval, and reuse of information. Reducing friction in long conversations and boosting creative productivity.

Research & Findings

We conducted extensive research to understand user needs and pain points with AI productivity tools.

Benchmarking

Analyzed competitor tools like Google Gemini, Claude, Grok and Microsoft Copilot to identify best practices and opportunities for differentiation.

User Interviews

Interviewed 30+ power users to understand their workflows and pain points with AI tools.

Usage Analytics

Analyzed usage data from 10,000+ users to identify common patterns and friction points.

Personas

Created 3 personas to represent the different user groups we interviewed.

Key Insight

Large context windows are limited and costly — users and AI alike struggle to retrieve relevant past information in long conversations. By structuring and surfacing key content through highlights, saved snippets, and a library, we reduce reliance on memory and help both sides maintain continuity.

The larger the window size, the more context the AI can use to generate a response. Learn more in https://artificialanalysis.ai/models

The Solution

We presented a redesigned ChatGPT interface that allows users to save, organize, and revisit key responses using features like snippet bookmarking, categorized folders, a quick-access bar, and note attachments. These tools were seamlessly integrated into the existing UI to enhance usability without overwhelming the user.

Highlighting option

Users can highlight text in the conversation to save it for later. The highlighted text is saved as a snippet and can be retrieved later.

Categorized folders in a Library

Users can categorize snippets into folders to organize them better. The folders are displayed in a sidebar and can be accessed quickly. The library is a searchable collection of all the snippets the user has saved.

Navigation Bar

A navigation bar in long conversations to quickly navigate to the previous messages.

Remixing snippets

Users can remix snippets to create new snippets. Creating an endless loop of creativity and productivity.

On the Highlighting Option screen, users can mark important parts of their conversation directly within the chat interface. Each highlight is visually distinct and stored for easy reference.

This solves the user's problem of losing key information in long chats by giving them a quick, effortless way to surface and revisit what matters most.

On the Quick Access Bar, users can instantly view their highlighted ideas from the current conversation, along with saved files and snippets from other chats—all in one side panel without leaving the conversation.

This solves the user's problem of constantly scrolling or switching tabs to retrieve context by keeping key information visible and easily accessible.

The Library Session allows users to organize and manage all their saved snippets in categorized folders, creating a structured knowledge base.

This solves the user's problem of losing track of valuable information over time by offering a structured, centralized space to store and retrieve insights efficiently.

The Snippet Remix feature allows users to select and combine saved highlights or snippets from different chats to generate new ideas or directions using AI.

This solves the user's problem of starting from scratch or losing creative momentum by sparking fresh ideation through previously saved insights.

Key Learnings

  • Context is power: Giving both users and AI better access to previous prompts can dramatically improve continuity and idea development.
  • Progressive disclosure: Showing advanced features only when needed kept the interface clean for beginners
  • Small UX changes = big impact: Small UX changes can have a big impact on user productivity, retention and satisfaction.
  • Flexibility is key: Allowing users to customize their dashboard view increased engagement significantly
